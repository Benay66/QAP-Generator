# QAP Instance Generator via Fourier Transform Decomposition

A generator of **Quadratic Assignment Problem (QAP)** instances with tunable difficulty, built on the Fourier Transform-based instance decomposition framework for permutation-based combinatorial optimization problems.

---

## Background

This project implements the instance generation method introduced in:

> **Benavides, X., Hernando, L., Ceberio, J., & Lozano, J. A.**
> *Fourier Transform-based instance decomposition for k-adic Assignment Problems.*
> IEEE Transactions on Evolutionary Computation, 2026.
> DOI: [10.1109/TEVC.2026.3674527](https://doi.org/10.1109/TEVC.2026.3674527)

The decomposition approach itself extends earlier work on the Linear Ordering Problem:

> **Benavides, X., Hernando, L., Ceberio, J., & Lozano, J. A.**
> *P-NP instance decomposition based on the Fourier transform for solving the Linear Ordering Problem.*
> Evolutionary Computation, MIT Press, 2025.

The decomposition code used and adapted in this project was originally developed by Xabier Benavides and is available at [Av-Repos/FT-Instance-Decomposition](https://github.com/Av-Repos/FT-Instance-Decomposition) (referenced in the paper above) and [Av-Repos/QAP-Decomposition](https://github.com/Av-Repos/QAP-Decomposition).

## What This Is

The **Quadratic Assignment Problem** (QAP) is a classical NP-Hard combinatorial optimization problem. Given a distance matrix `D` and a flow matrix `H`, the goal is to find a permutation `σ` of `{1, ..., n}` that minimizes:

```
f(σ) = Σᵢ Σⱼ d_{i,j} · h_{σ(i), σ(j)}
```

The difficulty of a QAP instance is known to depend on the *spectral structure* of its objective function — specifically, on the relative weight of the different Fourier coefficients of `f`. This generator exploits that relationship to produce instances with **controlled difficulty levels**.

### The Fourier decomposition of the QAP

The QAP is a particular case of the k-adic Assignment Problem (k-AP) with `k = 2`. The Fourier Transform on the symmetric group shows that the objective function of any QAP instance is entirely determined by four Fourier coefficients, associated with the partitions `(n)`, `(n-1,1)`, `(n-2,2)`, and `(n-2,1,1)`. Accordingly, any QAP flow matrix `H` can be decomposed as:

```
H = L⁽ⁿ⁾ + L⁽ⁿ⁻¹'¹⁾ + L⁽ⁿ⁻²'²⁾ + L⁽ⁿ⁻²'¹'¹⁾
```

where each sub-matrix `Lλ` captures the contribution of a single Fourier coefficient to the objective function.

- **Zero-order component** (`L⁽ⁿ⁾`): encodes the mean objective function value (constant shift, omitted in generation).
- **First-order component** (`L⁽ⁿ⁻¹'¹⁾`): captures univariate interactions — which item goes to which position. Instances dominated by this component are easier to optimize and, in limiting cases, reduce to a Linear Assignment Problem solvable in polynomial time.
- **Second-order components** (`L⁽ⁿ⁻²'²⁾`, `L⁽ⁿ⁻²'¹'¹⁾`): capture pairwise interactions between items. Increasing their weight makes the landscape more rugged and the problem harder.

### Instance generation

Given a source instance `(D, H)`, a new instance `(D, H_GEN)` is generated by linearly combining the sub-matrices:

```
H_GEN = (√β₁ / √Var[g¹]) · L⁽ⁿ⁻¹'¹⁾
       + (√(β₂/2) / √Var[g²ᵃ]) · L⁽ⁿ⁻²'²⁾
       + (√(β₂/2) / √Var[g²ᵇ]) · L⁽ⁿ⁻²'¹'¹⁾
```

The parameters `β₁` and `β₂ = 1 - β₁` directly control the **proportion of variance** explained by first- and second-order interactions respectively. By adjusting these weights, you can smoothly tune the difficulty of the generated instance from nearly-easy (first-order dominant) to NP-Hard-in-practice (second-order dominant).

## Features

- Exact Fourier decomposition of any QAP instance (`k = 2`) following the general k-AP framework.
- Exact variance computation for `n ≤ 8` (exhaustive enumeration) and accurate estimation for `n > 8`.
- Simple command-line interface.
- Output in **QAPLIB format**, compatible with standard QAP solvers and benchmarks.
- Reuses and adapts decomposition code originally developed by Xabier Benavides.

## Usage

```bash
python generator.py
```

The script will prompt you to:
1. Provide a source instance file (QAPLIB format).
2. Choose the weight parameters `β₁` (first-order) and `β₂` (second-order), with `β₁ + β₂ = 1`.
3. Specify an output file path.

**Example interaction:**

```
Source instance file: mi_instancia.dat
β₁ (first-order weight, 0–1): 0.3
Output file: instancia_generada.dat
```

This will generate an instance where 30% of the objective function variance comes from first-order interactions and 70% from second-order interactions — a relatively hard instance for local search.

### QAPLIB format

Input and output files follow the standard QAPLIB format:

```
n

d_{1,1}  d_{1,2}  ...  d_{1,n}
...
d_{n,1}  d_{n,2}  ...  d_{n,n}

h_{1,1}  h_{1,2}  ...  h_{1,n}
...
h_{n,1}  h_{n,2}  ...  h_{n,n}
```

## What to Expect

Empirical results from the paper show consistent trends across generated QAP instances:

- **Higher `β₂`** → lower autocorrelation coefficient → more rugged fitness landscape → more local optima → worse local search performance.
- **Higher `β₁`** → smoother landscape → fewer local optima → local search finds better solutions faster.
- At `β₁ = 1`, the instance is dominated by first-order information and becomes similar to a Linear Assignment Problem — solvable in polynomial time.
- At `β₂ = 1`, the instance captures only second-order interactions, and local search solutions can be as far from the global optimum as a random permutation.

This makes the generator useful for **benchmarking** optimization algorithms across a controlled difficulty spectrum, and for **stress-testing** heuristics in a principled way.
